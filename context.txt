Important runtime notes:
- Use `.venv/bin/python` for the sorter.
- Prefer year chunks and conservative worker counts. `--workers 6` and `--sleep 0.1` have been more stable than broader/faster runs.
- For interrupted all-years trash apply runs, rerun with `--resume` but without `--refresh-existing`.
- The all-years trash apply can run for hours because trash writes are single-message Gmail API calls.
- The sorter writes reports to `reports/`, checkpoints to `data/`, and manifests to `manifests/`.
- Trash stages use Gmail's Trash API, not permanent delete.
- Final all-years run should use `in:anywhere -in:trash` and a dedicated all-years progress file.
